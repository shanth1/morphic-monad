### Как теперь это тестировать (Пошаговый гайд)

Так как мы добавили ИИ для картинок (Vision) и Эмбеддинги, тебе нужно запустить локальный Ollama.

#### Шаг 1: Подготовка Ollama
В отдельном окне терминала убедись, что Ollama запущена и у неё скачаны нужные модели:
```bash
ollama run nomic-embed-text
ollama run llava
```
*(Если модели не скачаны, скачивание займет пару минут).*

#### Шаг 2: Запуск платформы
Очисти старый стейт и запусти TUI:
```bash
rm -rf ./tmp/nats-jetstream
go run cmd/forge/main.go
```
Выбери `config.local.yaml (Local AI Mode)`. Платформа стартанет.

---

#### Тест 1: Загрузка документа + SSE Стриминг (Chunker Worker)
Давай проверим, как разбивается на чанки длинный текст.

1. Создай текстовый файл `test.txt` с несколькими абзацами, разделенными пустой строкой (`\n\n`):
   ```text
   Первый абзац. Архитектура работает как часы.

   Второй абзац. Событийно-ориентированная система позволяет масштабироваться.
   ```
2. Отправь файл в Gateway (обрати внимание, теперь мы используем поле `files`):
   ```bash
   curl -X POST http://localhost:8080/v1/ingest \
     -H "X-Tenant-ID: tenant-1" \
     -F "files=@test.txt"
   ```
3. Ты получишь ответ:
   ```json
   {"correlation_id":"56f3a...","document_ids":["uuid-123..."],"status":"accepted"}
   ```
4. **Проверь SSE-стриминг!** Возьми `correlation_id` из ответа и вставь его в этот запрос (флаг `-N` нужен, чтобы curl не закрывал соединение):
   ```bash
   curl -N "http://localhost:8080/v1/events/stream?correlation_id=ТВОЙ_CORRELATION_ID"
   ```
   *Ты должен увидеть, как в реальном времени прилетают события от роутера, чанкера и эмбеддера.*

---

#### Тест 2: Загрузка картинки (Vision Worker)
Давай проверим мультимодальность. Найди любую картинку (например `cat.jpg` на рабочем столе) или создай пустую для теста.

1. Отправь картинку в Gateway:
   ```bash
   curl -X POST http://localhost:8080/v1/ingest \
     -H "X-Tenant-ID: tenant-1" \
     -F "files=@cat.jpg"
   ```
   *Что произойдет: Gateway поймет, что это `image/jpeg`. Роутер отправит это в Vision Worker. Ollama (модель llava) "посмотрит" на фото, сгенерирует текст "На фото кот". Затем этот текст уйдет в Embedder Worker и сохранится в VectorDB, НО с привязкой к оригинальному файлу `cat.jpg`.*

---

#### Тест 3: Мультимодальный Поиск
Теперь давай найдем то, что загрузили. Ты можешь искать просто текстом, или текстом + картинкой.

**Ищем текст:**
```bash
curl -X POST http://localhost:8080/v1/search \
  -H "X-Tenant-ID: tenant-1" \
  -F "query_text=масштабирование системы" \
  -F "top_k=2"
```

**Ответ будет выглядеть примерно так:**
```json
{
  "results": [
    {
      "document_id": "uuid-123...",
      "chunk_id": "uuid-456...",
      "score": 0.89,
      "text": "Второй абзац. Событийно-ориентированная система позволяет масштабироваться.",
      "file_uri": "memory://tenant-1/test.txt",
      "mime_type": "text/plain"
    }
  ]
}
```

#### Тест 4: Проверка Blob Proxy
Если в результатах поиска вернулся `file_uri` (например, `memory://tenant-1/cat.jpg`), фронтенд захочет его отрисовать.
Проверь работу нашего нового прокси:

```bash
curl -i "http://localhost:8080/v1/blob?uri=memory://tenant-1/test.txt"
```
